# Prompts Used During Development


## API Integration

- Explain how to use OpenWeather API with JavaScript.
- Show how to parse JSON responses and display data on the DOM.

## DOM Manipulation

- Update weather information dynamically using JavaScript.
- Implement city search functionality.
- Render weather icons from API responses.

## Error Handling

- Implement try/catch for API requests.
- Display user-friendly error messages for invalid city searches.
- Prevent application crashes during failed network requests.

## Geolocation

- Explain how to use navigator.geolocation.
- Fetch weather data based on user's current location.

## Local Storage

- Store weather data in LocalStorage.
- Implement 10-minute caching to reduce API requests.
- Save and retrieve user theme preferences.

## Dynamic Themes

- Change background themes based on weather conditions.
- Create sunny, cloudy, and rainy weather themes.
- Implement smooth theme transitions.

## Debugging & Optimization

- Fix API authorization issues.
- Resolve DOM rendering bugs.
- Improve code structure and maintainability.
- Optimize user experience and responsiveness.

## AI Usage Declaration

AI tools were used for learning, debugging, understanding API integration, and improving UI/UX design.

All project implementation, testing, deployment, and final integration were completed manually.
